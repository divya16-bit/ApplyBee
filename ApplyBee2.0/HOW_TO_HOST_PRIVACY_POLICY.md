# How to Get a Privacy Policy URL for Chrome Web Store

## Option 1: Host on Your Own Domain (Recommended)

If you have a website or domain:

### Steps:
1. **Upload the HTML file:**
   - Upload `PRIVACY_POLICY.html` to your website
   - Place it in the root directory or a `/privacy` folder

2. **Access via URL:**
   - Example: `https://yourdomain.com/privacy-policy.html`
   - Or: `https://yourdomain.com/privacy`

3. **Update the privacy policy:**
   - Replace `[DATE]` with current date (e.g., "January 1, 2025")
   - Replace `[YOUR_EMAIL@example.com]` with your contact email
   - Replace `[YOUR_WEBSITE_URL]` with your website URL
   - Replace `[YEAR]` with current year

---

## Option 2: Use GitHub Pages (Free)

### Steps:
1. **Create a GitHub repository:**
   - Go to GitHub.com
   - Create a new repository (e.g., "applybee-privacy")

2. **Upload the HTML file:**
   - Upload `PRIVACY_POLICY.html` to the repository
   - Rename it to `index.html` (optional, or keep as-is)

3. **Enable GitHub Pages:**
   - Go to repository Settings → Pages
   - Select source branch (usually `main`)
   - Save

4. **Get your URL:**
   - Your privacy policy will be at: `https://[your-username].github.io/applybee-privacy/privacy-policy.html`
   - Or create a custom domain if preferred

5. **Update the privacy policy file** with your actual information

---

## Option 3: Use Netlify (Free)

### Steps:
1. **Sign up:** Go to netlify.com (free account)

2. **Deploy:**
   - Drag and drop the folder containing `PRIVACY_POLICY.html`
   - Or connect to GitHub repository

3. **Get URL:**
   - Netlify provides: `https://[random-name].netlify.app/privacy-policy.html`
   - You can add a custom domain if you have one

4. **Update the privacy policy** with your information

## Option 4: Use Vercel (Free)

### Steps:

2. **Deploy:**
   - Import the project or drag & drop
   - Upload the HTML file

3. **Get URL:**
   - Vercel provides: `https://[project-name].vercel.app/privacy-policy.html`
   - Can add custom domain

## Option 5: Use Google Sites (Free & Easy)

### Steps:
1. **Go to:** sites.google.com

2. **Create a new site:**
   - Click "Create" → "New site"
   - Name it "ApplyBee Privacy Policy"

3. **Copy content:**
   - Copy the content from `PRIVACY_POLICY.html`
   - Paste into Google Sites
   - Format as needed

4. **Publish:**
   - Click "Publish"
   - Get URL: `https://sites.google.com/view/[your-site-name]`

5. **Update the privacy policy** content with your actual information

---

## Quick Setup Guide (Recommended: GitHub Pages)

### Step-by-Step:
1. **Create GitHub account** (if you don't have one)
2. **Create new repository:** `applybee-privacy`
3. **Upload `PRIVACY_POLICY.html`** to the repository
4. **Enable GitHub Pages:**
   - Settings → Pages
   - Source: `main` branch
   - Save
5. **Edit the HTML file** in GitHub:
   - Click the file → Edit (pencil icon)
   - Replace placeholders:
     - `[DATE]` → "January 1, 2025" (use today's date)
     - `[YOUR_EMAIL@example.com]` → your actual email
     - `[YOUR_WEBSITE_URL]` → your website (or leave as "#")
     - `[YEAR]` → "2025"
   - Commit changes
6. **Get your URL:** `https://[username].github.io/applybee-privacy/privacy-policy.html`

---

## Before Submitting to Chrome Web Store

### Checklist:
- [ ] Privacy policy is publicly accessible (no login required)
- [ ] URL is HTTPS (secure connection)
- [ ] All placeholders are replaced with actual information
- [ ] Privacy policy accurately describes what data you collect
- [ ] Contact information is included
- [ ] Last updated date is set
- [ ] Test the URL to ensure it loads correctly

---

## Example URLs:

✅ **Good examples:**
- `https://applybee.com/privacy-policy`
- `https://yourname.github.io/applybee-privacy`
- `https://applybee-privacy.netlify.app`

❌ **Bad examples:**
- `file:///C:/Users/...` (local file)
- `http://...` (not HTTPS)
- Password-protected page
- Google Drive link (not recommended)

---

## Tips:

1. **Keep it simple:** GitHub Pages is easiest and free
2. **Make it accessible:** Anyone should be able to view it without login
3. **Keep it updated:** Update the date when you make changes
4. **Test it:** Open the URL in an incognito window to ensure it's publicly accessible
